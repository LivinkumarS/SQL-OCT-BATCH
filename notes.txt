Constraints
--------------
>> Constraints are rules applied to a column to control what kind of data can be stored in it.

PRIMARY KEY --> unique, not null. it is used to identify the data uniquely. it can be added to only one column in a table.

FOREIGN KEY --> Used to link two tables. foreign key from a table will refer to primary key of another table.

AUTO_INCREMENT --> automatically generates a unique number.
UNIQUE --> ensures all values in a cloumn are different
NOT NULL --> no NULL values
DEFAULT --> Assign a default value when none is provided
CHECK (cond)--> It ensures the value meets specific condition

JOINS (Combining data from multiple tables)
------
>> A join combines rows from two or more tables on a related column between them.

INNER JOIN
------------
>> It returns only the records that have matching values in both tables.
Ex: SELECT (col details) FROM table1 INNER JOIN table2 ON table1.key=table2.key;

LEFT JOIN
----------
>> It returns all the records from left table, and the matched records from right table
Ex: SELECT (col details) FROM table1 LEFT JOIN table2 ON table1.key=table2.key;

RIGHT JOIN
----------
>> It returns all the records from right table, and the matched records from left table
Ex: SELECT (col details) FROM table1 RIGHT JOIN table2 ON table1.key=table2.key;

CROSS JOIN 
----------
>> It combines each row of the first table with each row from second table.
ex: select c.color, s.size from color c cross join size s;
